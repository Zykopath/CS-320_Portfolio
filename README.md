# CS-320 Portfolio Reflection

How can I ensure that my code, program, or software is functional and secure?

I can make sure that my software is functional and secure by following software engineering best practices throughout the development process. Unit testing is one of the most effective ways to verify that individual components perform as intended and continue to work correctly as changes are made. During this course, I created JUnit tests to validate the functionality of contact, task, and appointment services. Security and reliability are also improved by validating inputs, enforcing requirements, handling exceptions appropriately, and testing both normal and invalid use cases. By combining testing with code reviews and requirement verification, I can increase confidence that software behaves correctly and securely.

How do I interpret user needs and incorporate them into a program?

Interpreting user needs begins with carefully analyzing software requirements and translating them into functional features. Throughout this course, I reviewed project requirements to determine what constraints and behaviors each service needed to support. For example, contact IDs, names, phone numbers, and addresses had specific validation requirements that needed to be enforced within the application. By converting user and business requirements into classes, methods, and tests, I was able to ensure that the software met the intended expectations while providing a reliable user experience.

How do I approach designing software?

I approach software design by first identifying the requirements and breaking the problem into smaller, more manageable components. I focus on creating classes with clear responsibilities and designing solutions that are easy to maintain, test, and expand. In this course, I developed separate services and data classes for contacts, tasks, and appointments, which helped organize functionality and improve maintainability. I also used a test driven mindset by considering how features would be validated through unit tests. This approach helps ensure that the final product is structured, reliable, and aligned with user requirements.
